# 덱 레시피 (KO)

사용자가 이미 강한 발표 구조를 주지 않았다면 아래 기본 구조를 사용합니다.

## 문헌 리뷰
1. 제목과 맥락
2. 지금 이 주제가 왜 중요한지
3. 관련 논문 지형 또는 선정 기준
4. 핵심 논문: 문제와 아이디어
5. 방법 또는 구조
6. 메인 결과 figure 또는 table
7. 장점과 한계
8. 랩 관점 takeaway와 오픈 퀘스천

## 단일 논문 딥다이브
1. 제목, citation, 발표자
2. 동기와 문제 정의
3. 제안 방법
4. 핵심 figure 또는 system diagram
5. 메인 실험 또는 결과 table
6. 비판적 분석
7. 랩 후속 아이디어

## 연구 미팅
1. 제목과 현재 목표
2. 진행 현황 요약
3. 최신 결과, figure, 또는 failure case
4. 해석
5. 리스크나 blocker
6. 다음 단계
7. 필요하면 meeting log

## 레시피 규칙
- 사용자가 길게 요청하지 않으면 보통 5장부터 8장 사이를 기본으로 하세요.
- 소스가 받쳐주면 메인 result figure 또는 table 전용 슬라이드를 확보하세요.
- 학술 발표라면 비판과 next steps를 기본으로 포함하고, 사용자가 원할 때만 중립 요약만 유지하세요.
