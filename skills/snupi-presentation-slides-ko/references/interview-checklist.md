# SNUPI 연구 발표 인터뷰 체크리스트 (KO)

사용자가 Markdown 브리프, 노트, 논문 URL, PDF를 이미 준 경우에는 그 자료를 먼저 읽고, 정말 비어 있는 정보만 질문하세요.

## 기본 질문
1. 발표 종류가 무엇인가요: 문헌 리뷰, 단일 논문 딥다이브, 세미나 업데이트, 연구 미팅 중 어떤 형태인가요?
2. 청중은 누구이며, 어느 정도 기술적인 수준까지 다뤄야 하나요?
3. 이 발표의 한 문장 목표는 무엇인가요?
4. 이미 있는 자료는 무엇인가요: Markdown 노트, 논문 URL, PDF, 실험 결과, 미팅 기록 등?
5. 시간 제한이나 장수 제한 중 가장 중요한 제약은 무엇인가요?

## 연구 내용 질문
1. 반드시 포함해야 하는 논문이나 자료는 무엇인가요?
2. 가능하면 꼭 넣고 싶은 figure, table, equation이 있나요?
3. 청중이 마지막에 기억해야 할 주장이나 결과는 무엇인가요?
4. 한계점, 비판, 오픈 퀘스천도 포함해야 하나요?
5. 랩 차원의 해석, 후속 실험 아이디어, 액션 아이템이 있나요?

## 슬라이드 작성 메모
- 논문 전체를 요약하려 하지 말고 발표 스토리를 먼저 만드세요.
- 슬라이드마다 하나의 메시지와 하나의 큰 시각 요소를 우선하세요.
- 답이 outline이나 슬라이드 내용에 영향을 줄 때만 후속 질문을 하세요.
- 사용자가 이미 정리된 노트를 줬다면 처음부터 다시 인터뷰하지 말고 부족한 부분만 확인하세요.
