# Deck recipes (EN)

Use these as defaults when the user does not provide a strong deck structure already.

## Literature review
1. Title and context
2. Why this topic matters now
3. Paper landscape or selection criteria
4. Core paper: problem and idea
5. Method or architecture
6. Main results or figure
7. Strengths and limitations
8. Lab takeaways and open questions

## Single-paper deep-dive
1. Title, citation, presenter
2. Motivation and problem
3. Proposed method
4. Key figure or system diagram
5. Main experiment or table
6. Critical analysis
7. Follow-up ideas for the lab

## Research meeting
1. Title and current objective
2. Status snapshot
3. Latest result, figure, or failure case
4. Interpretation
5. Risks or blockers
6. Next steps
7. Optional meeting log

## Recipe rules
- Prefer 5 to 8 slides unless the user asks for a longer talk.
- Reserve one slide for the main result figure or table when the source material supports it.
- Include critique and next steps for academic audiences unless the user explicitly wants a neutral summary only.
